üìù Manual Entity Identification Process

1 Text Preprocessing
  • Tokenization: Break text into sentences and words
  • Normalization: Convert to lowercase, handle punctuation
  • Context Analysis: Consider surrounding words
2 Entity Type Identification
  • PROGRAM Entities: Look for "program", "initiative", "project"
  • AGENCY Entities: Look for "department", "agency", "bureau", acronyms
  • GOAL Entities: Look for percentages, years, targets
  • REPORTING Entities: Look for "report", "submit", "requirement"
  • STATUTE Entities: Look for "chapter", "section", "H.B.", legal citations
  • PURPOSE Entities: Look for infinitive verbs, objectives, goals
3 Context Validation
  • Surrounding Words: Check if context supports entity type
  • Grammatical Structure: Verify entity fits sentence structure
  • Domain Relevance: Ensure entity is relevant to legislative domain
4 Confidence Scoring
  • High Confidence (0.9): Exact pattern matches, clear context
  • Medium Confidence (0.7-0.8): Good context, some ambiguity
  • Low Confidence (0.5-0.6): Weak context, possible false positive

üéØ Best Practices for Manual Annotation

1. Consistency Rules

  • Always use full canonical names when possible
  • Maintain consistent capitalization
  • Use standard abbreviations consistently

2. Boundary Guidelines

  • Include articles when they're part of the entity name
  • Exclude punctuation unless it's part of the entity
  • Be consistent with compound entity boundaries

3. Context Considerations

  • Consider the legislative domain
  • Look for defining phrases ("the purpose of...")
  • Check for parallel structures in lists

4. Quality Checks

  • Verify entities make sense in context
  • Check for duplicate or overlapping entities
  • Ensure entity types are appropriate
